## :octicons-bug-16: Reporting a Bug 

!!! bug "Found something that doesn't look right?" 

    We welcome bug reports for issues occuring with:

    - **Data** — missing files, incorrect values, corrupted downloads
    - **Documentation** — broken links, outdated content, unclear instructions
    - **Code** — errors in notebooks, scripts, APIs, or tooling

If you have a GitHub account, please submit a *GitHub Issue* so the team can investigate and track the fix.

### How to submit an issue

1. Navigate to the [Issues tab](https://github.com/EO-DataHub/eodh-userdocs/issues) in the EO Data Hub repository.
2. Click *New issue*.
3. Choose the most relevant issue template (e.g. *Bug report*, *Data issue*), if one is available.
4. Fill in the title and description — the more detail you can provide, the easier it is for us to reproduce and fix the problem. Useful things to include:
   - A clear description of what you expected to happen and what actually happened
   - Steps to reproduce the issue
   - Relevant file names, dataset IDs, or URLs
   - Any error messages or screenshots
5. Click *Submit new issue*.

A member of the team will review your report and respond as soon as possible.

!!! info "No GitHub account?"

    Direct your queries to [enquiries@eodatahub.org.uk](mailto:enquiries@eodatahub.org.uk). If reporting a bug use the subject 'Reported bug', and include a clear description outlining steps taken that lead to the error being flagged or issue occurring, including screenshots. A member of the team will then raise an issue for you so that it can be tracked.
